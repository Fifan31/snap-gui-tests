Usage:
  gpt PixEx [options]

Description:
  Extracts pixels from given locations and source products.

Computed Properties:
PixExMeasurementReader measurements


Parameter Options:
  -PaggregatorStrategyType=<string>                  If the window size is larger than 1, this parameter describes by which m
ethod a single                                                     value shall be derived from the pixels.
                                                     Value must be one of 'min', 'max', 'mean', 'median', 'no aggregation'.
                                                     Default value is 'no aggregation'.
  -PcoordinatesFile=<file>                           Path to a file containing geo-coordinates. BEAM's placemark files can be used.
  -PdateInterpretationPattern=<string>               Describes how a date/time section inside a product filename should be interpreted. E.G. yyyyMMdd_hhmmss
                                                     Default value is 'yyyyMMdd'.
  -PexportBands=<boolean>                            Specifies if bands are to be exported
                                                     Default value is 'true'.
  -PexportExpressionResult=<boolean>                 If true, the expression result is exported per pixel, otherwise the expression
                                                     is used as filter (all pixels in given window must be valid).
                                                     Default value is 'true'.
  -PexportKmz=<boolean>                              If set to true, a Google KMZ file will be created, which contains the coordinates where pixels are found.
                                                     Default value is 'false'.
  -PexportMasks=<boolean>                            Specifies if masks are to be exported
                                                     Default value is 'true'.
  -PexportSubScenes=<boolean>                        If set to true, sub-scenes of the regions, where pixels are found, are exported.
                                                     Default value is 'false'.
  -PexportTiePoints=<boolean>                        Specifies if tie-points are to be exported
                                                     Default value is 'true'.
  -Pexpression=<string>                              Band maths expression (optional). Defines valid pixels.
  -PextractTimeFromFilename=<boolean>                If set to true, the sensing start and sensing stop should be extracted from the filename of each input product.
                                                     Default value is 'false'.
  -PfilenameInterpretationPattern=<string>           Describes how the filename of a product should be interpreted.
                                                     Default value is '*${startDate}*${endDate}*'.
  -PincludeOriginalInput=<boolean>                   Determines if the original input measurements shall be included in the output.
                                                     Default value is 'false'.
  -PmatchupFile=<file>                               Path to a CSV-file containing geo-coordinates associated with measurements accordingto BEAM CSV format specification
  -PoutputDir=<file>                                 The output directory.
                                                     This is a mandatory parameter.
  -PoutputFilePrefix=<string>                        The prefix is used to name the output files.
                                                     Default value is 'pixEx'.
  -PsourceProductPaths=<string,string,string,...>    A comma-separated list of file paths specifying the source products.
                                                     Each path may contain the wildcards '**' (matches recursively any directory),
                                                     '*' (matches any character sequence in path names) and
                                                     '?' (matches any single character).
  -PsubSceneBorderSize=<int>                         An additional border around the region where pixels are found.
                                                     Default value is '0'.
  -PtimeDifference=<string>                          The acceptable time difference compared to the time given for a coordinate.
                                                     The format is a number followed by (D)ay, (H)our or (M)inute. If no time
 difference is provided, all input products are considered regardless of their time.
  -PwindowSize=<integer>                             Side length of surrounding window (uneven)
                                                     Default value is '1'.
